- 입지 적합도
- 영업 유지 proxy 점수
- 상가정보 스냅샷상 관측 유지
- 공공데이터 기반 후보지 점수
- 의사결정 지원 도구
- 1차 후보지 필터링

## 6. 서론 초안

소상공인은 지역경제를 구성하는 핵심 주체이지만, 창업 입지와 업종 선택 과정에서 충분한 데이터 분석을 활용하기 어렵다. 중소벤처기업부와 소상공인시장진흥공단의 조사에 따르면 2024년 기준 소상공인 기업체는 613만 개, 종사자는 961만 명에 이른다.[^1] 이처럼 소상공인은 고용과 지역경제에서 큰 비중을 차지하지만, 실제 창업 과정에서는 임대료, 유동인구, 경쟁 업종 분포, 교통 접근성 등을 종합적으로 비교하기 어렵다. 특히 개별 창업자는 전문 상권 분석 도구를 활용하기보다 경험이나 직관에 의존하는 경우가 많다.

최근에는 공공데이터를 활용한 상권 분석 연구와 서비스가 확대되고 있다. 공공데이터포털과 소상공인365는 업종 현황, 상권 정보, 유동인구, 매출 관련 통계 등 다양한 자료를 제공한다. 그러나 이러한 서비스는 사용자가 직접 여러 지표를 해석해야 하며, 특정 위치와 업종을 함께 고려한 후보지 비교나 설명 제공에는 한계가 있다. 소상공인365 서비스 소개에서도 행정동 단위의 “업종 분포, 유동인구” 자료를 제공한다고 설명하지만,[^2] 실제 창업자는 이러한 지표가 왜 특정 입지 판단에 영향을 미치는지 추가 해석을 필요로 한다.

이에 본 연구는 광주광역시를 대상으로 공공데이터와 설명가능한 AI를 결합한 소상공인 입지 분석 시스템을 설계하고 구현한 과정을 검토한다. 본 연구의 목적은 실제 창업 결과를 단정하는 것이 아니라, 광주 지역의 상가정보 스냅샷, 교통 접근성, 업종 밀집도, 비용 부담 proxy 등을 활용하여 입지 후보지를 1차로 비교할 수 있는 의사결정 지원 구조를 제시하는 데 있다. 특히 시스템은 입지 적합도 점수뿐 아니라 추천 사유와 주의 사유를 함께 제공함으로써 사용자가 점수의 의미와 한계를 이해할 수 있도록 설계되었다.

## 7. 본론 초안

### 7.1 소상공인 입지 의사결정과 공공데이터의 필요성

소상공인의 입지 선택은 단순히 점포 위치를 고르는 문제가 아니라 수요, 경쟁, 접근성, 비용을 동시에 고려해야 하는 복합적인 의사결정이다. 김영갑과 상권분석전략연구회는 상권 분석에서 유동인구, 배후인구, 업종 밀집도, 임대료 수준 등을 핵심 변수로 제시한다.[^3] 이는 소상공인 입지 선택이 경험적 감각만으로 판단하기 어려운 데이터 기반 문제임을 보여준다.

공공데이터 기반 접근은 이러한 문제를 완화할 수 있다. 공공데이터포털은 전국 상가정보와 지역별 상권 관련 자료를 제공하고, 소상공인365는 상권의 업종 구성과 인구·소비 관련 지표를 조회할 수 있도록 한다. 이러한 자료는 개별 창업자가 직접 수집하기 어려운 정보를 비교적 낮은 비용으로 활용할 수 있게 한다는 장점이 있다. 다만 여러 데이터가 서로 다른 기준과 단위로 제공되기 때문에, 실제 시스템에서는 좌표 정제, 행정구역 매칭, 업종 분류 통합, 결측치 처리 같은 전처리 과정이 필요하다.

### 7.2 선행연구와 기존 서비스의 한계

장호준·이기환·이희상은 공공데이터와 설명가능한 AI를 결합하여 자영업 상권 분석과 장기 관측 유지 관련 분류 모델을 제시하였다. 이 연구는 상가정보, 지리적 요인, 인구통계 요인을 함께 고려했다는 점에서 공공데이터 기반 상권 분석의 가능성을 보여준다.[^4] 특히 해당 연구는 자영업이 지역경제에서 “중요한 역할”을 한다고 설명하며, 소상공인 데이터 분석의 사회적 필요성을 강조한다.[^5]

김철민의 연구도 지역 기반 상권 데이터와 머신러닝을 결합했다는 점에서 참고할 수 있다. 이 연구는 수원시 근린상권을 대상으로 “XGBoost 머신러닝 알고리즘”을 활용하여 지역 단위 상권 지표를 분석하였다.[^6] 이러한 선행연구들은 공공데이터와 머신러닝이 상권 분석에 활용될 수 있음을 보여주지만, 서울과 수도권 사례에 집중되어 있다는 한계가 있다. 따라서 광주광역시와 같은 비수도권 광역시에 맞춘 지역 단위 시스템 설계 연구가 필요하다.

기존 상권정보 서비스 역시 유용한 기초 통계를 제공하지만, 사용자가 직접 여러 지표를 해석해야 한다. 예를 들어 특정 업종의 밀집도가 높다는 사실은 수요가 존재한다는 신호일 수도 있지만, 동시에 경쟁 부담이 크다는 뜻일 수도 있다. 따라서 시스템은 단순히 점수만 제시해서는 안 되며, 동일한 지표가 가질 수 있는 복수의 해석을 함께 보여주어야 한다.

### 7.3 BizSpot AI 시스템 설계

본 연구에서 설계한 BizSpot AI는 광주광역시 소상공인 입지 검토를 위한 웹 기반 의사결정 지원 시스템이다. 시스템은 React/Vite 기반 프론트엔드와 Node Express 및 Vercel serverless API 기반 AI 설명 기능으로 구성되었다. 지도 화면은 Kakao Maps SDK를 우선 사용하되, API key가 없거나 로딩에 실패할 경우 광주 bounding box 기반 2D fallback map으로 동작하도록 설계하였다. 이를 통해 실제 지도 연동이 불안정한 상황에서도 최소한의 클릭 분석 기능을 유지할 수 있다.

데이터 측면에서는 `candidates_balanced.json`과 `grid_scores.json`을 핵심 웹 데이터로 사용하였다. `candidates_balanced.json`은 업종별 후보지 표시와 AI 상담 TOP 3 후보 선택에 사용되며, `grid_scores.json`은 사용자가 지도에서 임의의 위치를 클릭했을 때 가장 가까운 500m 상권 격자를 찾아 분석하는 데 사용된다. `grid_scores.json` 전체를 지도 마커로 표시하면 화면이 복잡해지고 성능이 저하될 수 있으므로, 지도에는 제한된 수의 후보지와 선택 업종 상위 격자만 표시하고, 격자 데이터는 클릭 분석용 lookup 데이터로 활용하였다.

점수 체계는 입지 적합도와 영업 유지 proxy 점수를 분리하였다. 입지 적합도는 수요 proxy, 교통 접근성, 업종 궁합, 경쟁 부담, 비용 부담 proxy를 조합한 웹용 종합 점수이다. 반면 영업 유지 proxy 점수는 상가정보 스냅샷상 관측 유지 패턴을 기반으로 한 보조 지표이다. 두 점수는 같은 산식으로 만들지 않았으며, 실제 창업 결과를 단정하는 표현 대신 후보지 비교를 위한 참고 지표로 제시하였다.

### 7.4 실험 및 구현 결과

현재 웹 구현에서는 총 160개의 균형 후보지와 3,454개의 업종별 격자 점수 row가 생성되었다. 500m 기준 base grid는 781개이며, 사용자 표시 업종은 카페, 디저트·베이커리, 일반 음식점, 분식, 치킨, 편의점, 미용실, 세탁소의 8개로 제한하였다. 좌표가 없거나 광주 범위를 벗어난 후보는 제외하였고, 사용자 표시용 데이터에는 `other` 업종을 포함하지 않았다.

지도 클릭 분석은 grid-first 방식으로 구현되었다. 사용자가 지도 또는 fallback map을 클릭하면 해당 좌표와 가장 가까운 `base_grid_id`를 먼저 찾고, 선택 업종 row가 있으면 해당 row를 분석 결과로 표시한다. 선택 업종 row가 없을 경우 같은 격자 내 입지 적합도 상위 5개 업종 중 1위를 대표 결과로 보여준다. 격자 데이터가 없는 경우에는 기존 후보지 기반 nearest candidate fallback을 사용한다. 이 방식은 사용자가 지도 아무 곳이나 클릭했을 때도 “그 위치 주변의 분석 결과”에 가까운 경험을 제공한다.

AI 상담 기능은 사용자의 질문에서 지역과 업종 키워드를 추출한 뒤 로컬 JSON에서 TOP 3 후보를 먼저 선택한다. 이후 서버 API는 이 TOP 3 후보를 바탕으로 자연어 설명만 생성한다. 즉 AI가 새 후보지나 새 좌표를 임의로 생성하지 않도록 구조를 제한하였다. 실제 테스트에서는 로컬 Express API와 Vercel serverless API 모두 `gemma-4-31b-it` 모델을 통해 응답을 반환하는 것을 확인하였다. 또한 모델 응답 중 내부 사고 과정으로 표시될 수 있는 `thought` part는 최종 사용자 화면에 표시되지 않도록 필터링하였다.

### 7.5 설명가능성과 한계

BizSpot AI의 핵심은 단순 점수 표시가 아니라 설명가능성이다. 우측 분석 패널은 입지 적합도, 영업 유지 proxy 점수, 위험도, 선택 업종, 추천 사유, 주의 사유, 그리고 해당 격자의 추천 업종 TOP 5를 함께 표시한다. 특히 동일 업종 밀집도는 수요 신호이면서 동시에 경쟁 부담일 수 있으므로, 추천 사유와 주의 사유에서 두 해석을 함께 제시하도록 설계하였다.

그러나 본 시스템에는 명확한 한계가 있다. 첫째, `grid_scores.json`은 광주 행정구역 전체의 모든 빈 격자를 생성한 자료가 아니라 상가 데이터가 존재하는 commercial grid 중심의 분석 자료이다. 따라서 빈 지역을 클릭하면 가장 가까운 분석 격자 또는 후보지를 참고로 표시한다. 둘째, 비용 관련 변수는 실제 월세가 아니라 공시지가와 실거래가 계열 자료를 활용한 비용 부담 proxy이다. 셋째, 본 시스템은 실시간 유동인구나 점포 내부 상태, 권리금, 브랜드 전략 같은 현장 정보를 포함하지 못한다. 따라서 최종 창업 판단에는 현장 조사와 추가 데이터가 반드시 필요하다.

## 8. 결론 초안

본 연구는 광주광역시 소상공인 입지 검토를 지원하기 위해 공공데이터와 설명가능한 AI를 결합한 웹 기반 상권 분석 시스템을 설계하고 구현하였다. 시스템은 상가정보와 교통 접근성, 업종 밀집도, 비용 부담 proxy 등을 활용하여 후보지의 입지 적합도와 영업 유지 proxy 점수를 제시하며, 지도 클릭 분석과 AI 상담 기능을 통해 사용자가 후보지를 비교할 수 있도록 구성되었다.

연구 결과, 공공데이터만으로도 광주 지역 상권 후보지를 일정 수준으로 구조화하고 비교하는 시스템을 설계할 수 있음을 확인하였다. 특히 500m 격자 기반 분석과 후보지 fallback 구조는 사용자가 특정 좌표를 클릭했을 때 주변 상권을 직관적으로 검토할 수 있게 한다. 또한 AI 상담 기능을 로컬 JSON 기반 TOP 3 후보 설명으로 제한함으로써, AI가 임의의 후보지를 생성하는 문제를 줄이고 설명 도구로 활용할 수 있었다.

다만 본 시스템은 공공데이터 기반 proxy 분석에 머문다는 한계가 있다. 실제 창업 판단에는 임대료, 권리금, 점포 내부 상태, 브랜드 전략, 실제 유동인구 등 현장 정보가 추가로 필요하다. 향후 연구에서는 더 정밀한 공간 검증, 시간 기반 검증, 사용자 평가, 실제 운영 데이터와의 비교를 통해 시스템의 신뢰도를 높일 필요가 있다.

## 9. 직접 인용 후보

직접 인용은 반드시 짧게 쓰고, 각주에 출처를 명확히 달아야 한다. 아래 문장은 그대로 사용 가능하되, 워드에서 각주 번호를 붙이면 된다.

### 직접 인용 후보 1: 소상공인 규모

본문 예시:

> 중소벤처기업부와 소상공인시장진흥공단은 2024년 기준 “소상공인 기업체는 613만 개”라고 발표하였다.[^1]

사용 위치:

- 서론 첫 문단
- 연구 필요성 설명

### 직접 인용 후보 2: 공공 상권 데이터 항목

본문 예시:

> 소상공인365는 상권 분석을 위해 “업종 분포, 유동인구” 등의 자료를 제공한다.[^2]

사용 위치:

- 서론 또는 본론 2.1
- 기존 서비스 설명

### 직접 인용 후보 3: 자영업의 사회적 중요성

본문 예시:

> 장호준 외는 자영업이 지역경제에서 “중요한 역할”을 한다고 설명한다.[^5]

사용 위치:

- 선행연구 검토

### 직접 인용 후보 4: 머신러닝 방법

본문 예시:

> 김철민의 연구는 “XGBoost 머신러닝 알고리즘”을 활용한 지역 단위 상권 분석 사례이다.[^6]

사용 위치:

- 선행연구 검토
- 방법론 비교

### 주의

김영갑·상권분석전략연구회 단행본은 실제 책 페이지를 확인한 뒤 직접 인용을 넣는 것이 안전하다. 페이지를 확인하지 못하면 직접 인용보다 간접 인용으로 처리한다.

## 10. 간접 인용 후보

간접 인용은 원문의 뜻을 자신의 문장으로 바꾸어 쓰는 방식이다.

### 간접 인용 후보 1

김영갑과 상권분석전략연구회는 상권 분석에서 유동인구, 배후인구, 경쟁 업종, 임대료 등의 변수를 종합적으로 검토해야 한다고 설명한다.[^3]

### 간접 인용 후보 2

장호준·이기환·이희상의 연구는 공공데이터와 설명가능한 AI가 자영업 상권 분석에 활용될 수 있음을 보여준 사례로, 본 연구의 시스템 설계 방향에 중요한 선행연구가 된다.[^4]

### 간접 인용 후보 3

김철민의 연구는 지역 단위 상권 데이터를 활용한 머신러닝 분석이 가능하다는 점을 보여주지만, 수도권 사례에 기반하고 있어 광주광역시와 같은 비수도권 지역에 대한 추가 적용이 필요하다.[^6]

### 간접 인용 후보 4

소상공인365와 공공데이터포털은 상권 분석을 위한 기초 자료를 제공하지만, 사용자가 여러 지표를 직접 해석해야 한다는 점에서 설명형 의사결정 지원 시스템의 필요성이 남아 있다.[^2]

본 논문 초고는 시카고 양식의 각주-참고문헌 규정에 따라 작성한다. 본문에서 참고자료를 활용한 부분은 각주로 출처를 제시하고, 글 마지막에는 저자명 가나다순에 가까운 방식으로 참고문헌을 정리한다. 직접 인용은 따옴표로 표시하며, 간접 인용은 원문의 내용을 연구자의 문장으로 바꾸어 서술하되 각주를 통해 출처를 밝힌다.
